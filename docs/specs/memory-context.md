# Spec: Memory & Context

## Типы памяти

### 1. Short-term Memory (in-process)

**Что:** Диалоговая история текущей сессии (≤10 сообщений).

**Где:** Python dict в памяти процесса; ключ = `user_hash`.

**Формат:**
```python
conversation_history: list[dict] = [
    {"role": "user",      "content": "загрузи файл"},
    {"role": "assistant", "content": "✅ Загружено 47 транзакций"},
    {"role": "user",      "content": "лимит 300 на кофе"},
    ...
]
```

**Политика:** Скользящее окно — при добавлении 11-го сообщения удаляется первое (не системный промпт).

**Сброс:** При рестарте процесса история теряется. Пользователь видит: "Привет! Я перезапустился. Можешь снова загрузить файл."

---

### 2. Long-term Memory (persistent)

**Что:** Транзакции, лимиты, pending confirmations.

**Где:** JSON-файл `/data/sessions/{user_hash}.json`.

**Формат:**
```json
{
  "user_hash": "a3f8b2...",
  "created_at": "2026-03-01T10:00:00Z",
  "last_active": "2026-03-28T14:30:00Z",
  "transactions": [
    {"date": "2026-03-15", "amount": 450.0, "merchant": "Starbucks", "category": "Кофе"}
  ],
  "limits": {
    "Кофе": {"amount": 300.0, "period": "week", "set_at": "2026-03-28T10:15:00Z"}
  },
  "pending_confirmation": null
}
```

**TTL:** Файлы с `last_active` старше 7 дней удаляются фоновым cron-job'ом.

**Удаление по команде `/delete`:** немедленно, без recovery.

---

### 3. Pending Confirmation State

Когда confidence NL-парсинга < 0.8, агент не применяет лимит автоматически — сохраняет `pending_confirmation`:

```json
{
  "pending_confirmation": {
    "type": "set_limit",
    "params": {"category": "Кофе", "amount": 300, "period": "week"},
    "expires_at": "2026-03-28T10:25:00Z",
    "message_id": 12345
  }
}
```

**TTL:** 10 минут. После истечения — автоматически отменяется.

## Context Budget (LLM)

| Слот | Токены | Содержимое |
|------|--------|-----------|
| System prompt | ~500 | Роль агента, правила, формат вывода, список инструментов |
| User data summary | ≤1500 | Агрегаты: топ-5 категорий за период, текущие лимиты, нарушения |
| Conversation history | ≤1000 | Последние 10 сообщений (короткие) |
| Current request | ≤500 | Текущий запрос пользователя |
| **Итого input** | **≤3500** | Безопасный потолок; оставляет место для ответа модели |
| Reserved for output | ~500 | Ответ модели |

**Правило:** Сырые транзакции (тысячи строк) в LLM **не передаются никогда**. Только агрегаты.

**Пример "user data summary":**
```
Загружено: 47 транзакций за март 2026
Топ-категории:
  Продукты: 8500₽ (35%)
  Кофе: 2100₽ (9%) ⚠️ превышает лимит 300₽/нед
  Такси: 1800₽ (7%)
Активные лимиты: Кофе 300₽/нед
```

---

## Memory Policy

| Правило | Почему |
|---------|--------|
| PII не хранится | Privacy-by-design |
| Telegram ID только как хэш | Невозможно восстановить оригинал |
| История ≤10 сообщений | Контроль token budget |
| Файл транзакций удаляется сразу после парсинга | Минимизация хранения чувствительных данных |
| TTL 7 дней | Баланс UX (не надо каждый раз загружать) и приватности |
| `/delete` — немедленное удаление | Соответствие ожиданиям пользователя |
| Cross-user изоляция по user_hash | Пользователи не видят данные друг друга |
