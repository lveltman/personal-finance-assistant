# Spec: Retriever / Categorizer

## Назначение

Определить финансовую категорию транзакции по названию merchant'а и описанию.
Возвращает: `(category: str, confidence: float, method: str)`.

## Архитектура: 2-уровневый каскад (PoC)

> **Примечание:** В PoC реализованы уровни 1 и 3. Уровень 2 (embedding) — запланирован для следующей фазы.

```
Транзакция: merchant="STARBUCKS #1234", amount=450
         ↓
Level 1: Keyword Match
  • Статичный словарь: {"starbucks": "Кофе", "mcdonalds": "Фастфуд", ...}
  • ~100 записей в data/merchant_rules.json
  • Точное совпадение → confidence 1.0; подстрока → confidence 0.9
  • Latency: < 1ms
  • Coverage: ~70% транзакций
         ↓ нет совпадений
Level 3: LLM Fallback (реализован)
  • Промпт: "Определи категорию для мерчанта: '{merchant}'. Ответь одним словом из списка: {categories}"
  • Fallback: категория "Прочее" если LLM недоступен или не распознал
  • Coverage: оставшиеся ~30%

--- (запланировано, не реализовано в PoC) ---
Level 2: Embedding Similarity
  • Модель: sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
  • Индекс: 50 категорий × {название + описание + примеры}
  • Метрика: cosine similarity; порог ≥ 0.75
  • Ожидаемая coverage: ~20% оставшихся (снизит нагрузку на LLM)
```

## Источники и индекс

**Статичный словарь** (`data/merchant_rules.json`):
```json
{
  "starbucks": "Кофе",
  "coffee": "Кофе",
  "kfc": "Еда фастфуд",
  "аптека": "Здоровье",
  ...
}
```

**Категории (50 шт.):**
Продукты, Кофе, Рестораны, Транспорт, Такси, Здоровье, Фитнес, Одежда, Электроника, Развлечения, Путешествия, Связь, Коммунальные, Образование, Другое, ...

**Embedding индекс** (запланировано, не реализовано в PoC):
- Размер: ~50 векторов × 384 измерения ≈ 75 KB
- Инициализация: < 2s при старте

## Reranking

В PoC reranking не требуется — каскад уже упорядочен по точности.
Если оба embedding и LLM дают разные ответы → приоритет у embedding (быстрее, детерминированнее).

## Ограничения

| Параметр | Значение |
|----------|----------|
| Точность (F1) | ≥ 0.92 на тестовом наборе |
| Latency p95 | < 50ms (keyword + embedding); < 10s (LLM fallback) |
| Поддерживаемые языки | RU, EN |
| Размер батча | 1 транзакция (PoC); батчинг — фаза 2 |
| Обновление словаря | Вручную через PR |
| Нераспознанные | Категория "Другое" + флаг для ручной разметки |

## Тестирование

- Golden set: 200 транзакций с ручной разметкой
- Автотест: `pytest tests/test_categorizer.py` — F1 ≥ 0.92
- Edge cases: транзакции на кириллице, латинице, смешанные, без merchant name
