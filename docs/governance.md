| **Prompt injection через пользовательский ввод** | 🟠 Высокая / 🟠 Среднее | - Тесты на adversarial prompts в CI<br>- Мониторинг аномальных паттернов в логах<br>- Rate limiting на запросы | - Санитизация ввода: regex-фильтры на `__import__`, `eval`, `; rm`, `DROP TABLE`<br>- Изоляция инструментов: sandbox для внешних вызовов<br>- Контекст-ограничение: системный промпт фиксирует роль «только финансовый анализ»<br>- Подтверждение критических действий (см. раздел 3) | 🟢 Низкий |
| **Некорректная категоризация расходов** | 🔴 Высокая / 🟢 Низкое | - Метрика F1 на валидационном наборе<br>- User feedback: «Это не та категория»<br>- Аномалии: резкие скачки в категории | - Hybrid-подход: rules-based fallback + ML<br>- Возможность ручной правки с сохранением в user profile<br>- Объяснение в отчёте: «Категория определена как X, потому что...» | 🟢 Низкий |
| **Зависимость от внешних API (цены, курсы)** | 🟡 Средняя / 🟡 Среднее | - Health-check эндпоинтов перед вызовом<br>- Логирование таймаутов/ошибок<br>- Мониторинг freshness данных | - Кэширование ответов (TTL: 1 час для цен, 15 мин для курсов)<br>- Fallback на офлайн-базу (последние известные значения)<br>- Graceful degradation: «Данные устарели, показываем оценку» + warning в отчёте | 🟡 Средний |
| **Превышение ресурсов (OOM, latency) на больших файлах** | 🟡 Средняя / 🟠 Среднее | - Мониторинг memory/CPU в Docker<br>- Логирование времени обработки<br>- Alert при >90% лимита | - Лимит строк по умолчанию: `MAX_FILE_ROWS=10000`<br>- Потоковая обработка: chunking по 1000 строк<br>- Ограничение ресурсов в docker-compose: `memory: 2G, cpus: '2'`<br>- Прогресс-бар для UX при долгой обработке | 🟢 Низкий |
| **Несоответствие регуляторным требованиям (GDPR/152-ФЗ)** | 🟢 Низкая / 🔴 Высокое | - Юридический review перед релизом<br>- Чек-лист соответствия в CI/CD<br>- Аудит хранения данных | - Data minimization: собираем только необходимые поля<br>- Право на удаление: команда `--purge` удаляет все артефакты<br>- При деплое в РФ: хранение на территории РФ, уведомление РКН при необходимости<br>- DPA-шаблон готов к подписанию | 🟢 Низкий |

## 2. 🔐 Политика логирования и работы с персональными данными

### 2.1 Принципы обработки данных
```yaml
data_handling_principles:
  - local_first: true          # По умолчанию — обработка на устройстве пользователя
  - cloud_sync: opt_in         # Облачная синхронизация только при явном согласии
  - minimization: true         # Собираем только: дата, сумма, категория, комментарий (опц.)
  - retention_default: 7d      # Логи агента хранятся 7 дней, затем автоудаление
  - right_to_delete: true      # Команда --purge удаляет все данные без восстановления
```

### 2.2 Классификация данных
| Тип данных | Пример | Обработка | Хранение |
|------------|--------|-----------|----------|
| **PII (персональные)** | Имя, email, номер карты | Маскировка/хэширование при логировании | Не хранится после сессии (по умолчанию) |
| **Финансовые транзакции** | Дата, сумма, магазин, категория | Анализ в памяти, экспорт по запросу | Временные файлы в `/output` — удаляются пользователем |
| **Настройки лимитов** | «≤500₽ на кофе/неделю» | Сохранение в локальный профиль (опционально) | Локальный JSON, шифруется при наличии пароля |
| **Агентские логи** | Трейсы рассуждений, вызовы инструментов | Анонимизация user_id (SHA-256 + salt) | 7 дней, затем ротация |

### 2.3 Логирование: структура и уровни
```python
# Пример структурированного лога (JSON)
{
  "timestamp": "2026-03-05T14:30:00Z",
  "session_id": "sha256:a1b2c3...",  # хэш, не обратимый
  "event_type": "limit_set",
  "input": {
    "nl_query": "не тратить больше 300 на кофе в неделю",
    "file_hash": "sha256:xyz789..."  # только хэш файла, не контент
  },
  "agent": {
    "parsed_params": {"category": "coffee", "amount": 300, "period": "week"},
    "confidence": 0.94,
    "tools_used": ["nl_parser", "limit_validator"],
    "latency_ms": 1240
  },
  "output": "Лимит установлен. Предыдущие траты: 420₽ → превышение на 40%",
  "sensitivity": "low"  # low/medium/high — для фильтрации при экспорте
}
```

**Уровни логирования:**
| Уровень | Что логируется | Когда использовать |
|---------|---------------|-------------------|
| `ERROR` | Исключения, сбои инструментов, validation errors | Всегда включён |
| `WARNING` | Превышение лимитов, неопределённые категории, fallback'и | По умолчанию |
| `INFO` | Успешные действия: загрузка файла, установка лимита, экспорт | Dev / при отладке |
| `DEBUG` | Полные трейсы LLM, сырые ответы API, internal state | Только local dev, никогда в prod |

### 2.4 Соответствие регуляторам
- **GDPR (ЕС)**: 
  - Право на доступ: команда `--export-raw` возвращает все данные пользователя в машиночитаемом формате
  - Право на исправление: ручная правка категории обновляет профиль и пере-анализирует историю
  - DPO contact: [заполняется при коммерческом использовании]

- **152-ФЗ (РФ)**:
  - При деплое на территории РФ: все данные хранятся в пределах РФ
  - Уведомление Роскомнадзора — при сборе PII в коммерческом продукте
  - Шифрование персональных данных при передаче (TLS 1.3+) и хранении (AES-256)

- **PCI DSS** (если обрабатываются данные карт):
  - ❌ Не принимаем полные номера карт, CVV, сроки
  - ✅ Если пользователь вставил выписку с картой — автоматически маскируем: `**** **** **** 1234`
  - ✅ Не передаём данные карт во внешние API

| Действие | Требует подтверждения? | Механизм подтверждения | Пример UX |
|----------|------------------------|------------------------|-----------|
| **Отправка запроса на возврат** | ✅ Да | Двухэтапный flow + явная кнопка | «Найдена возможность возврата 420₽ за кофе. Отправить запрос в магазин [Brand X]?» → [✅ Отправить] / [❌ Отмена] |
| **Сохранение лимита в профиль** | ✅ Да | Чекбокс + пояснение | «Сохранить правило «≤300₽ на кофе/неделю» для будущих анализов?» → [✅ Да, сохранять] / [❌ Только сейчас] |
| **Экспорт отчёта с PII** | ✅ Да | Выбор уровня анонимизации | «Экспортировать отчёт:» → [🔓 С полными данными] / [🔐 Только анонимизированный] / [❌ Отмена] |

```json
    "security": {

      "pii_detected": false,
      "confidence_threshold_met": true
    }
  }

```

**Политика хранения аудита:**
- 🔹 Логи аудита хранятся **7 дней** в локальной папке `./logs/audit/`
- 🔹 Автоматическая ротация: старые логи архивируются и удаляются
- 🔹 Доступ к логам: только владелец сессии (через локальный доступ к файлам)
- 🔹 При экспорте для отладки: обязательная анонимизация перед отправкой

## 4. 🧭 Этические принципы и прозрачность

### 4.1 Принципы AI-ассистента
```yaml
ethical_guidelines:
  transparency:
    - Каждая рекомендация сопровождается «почему»: источник данных + логика
    - Пользователь видит confidence-score и может запросить альтернативу
  
  user_control:
    - Режим «только калькулятор»: отключение AI-рекомендаций одной кнопкой
    - Полный экспорт и удаление данных по запросу
  
  no_dark_patterns:
    - Не навязываем платные услуги или партнёрские предложения
    - Приоритет: экономия пользователя > монетизация
  
  fairness:
    - Тестирование категоризации на разнообразных демографических данных
    - Мониторинг bias: не дискриминируем по типу расходов, локации, брендам
```

### 4.2 Объяснимость рекомендаций (XAI-lite)
Каждая рекомендация в отчёте включает блок «Почему это предложено»:

```markdown
> 💡 Рекомендация: Попробуйте бренд «Z» вместо «Starbucks» — экономия 130₽
> 
> **Обоснование:**
> - Вы потратили 420₽ на кофе в Starbucks 03.03.2026
> - Аналогичный продукт (латте 400мл) в бренде «Z»: 290₽ [источник: PriceAPI, 05.03 14:20]
> - Разница: 130₽ (31% экономии)
> - Ближайшая точка «Z»: ул. Примерная, 15 (1.2 км от вашей последней покупки)
> 
> **Уверенность системы:** 91%  
> **Что проверить:** Актуальность цены, наличие товара
```
